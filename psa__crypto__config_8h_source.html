11// <o SL_PSA_KEY_USER_SLOT_COUNT> PSA User Maximum Open Keys Count <0-128>
12// <i> Maximum amount of keys that the user application will have open
13// <i> simultaneously. In context of PSA Crypto, an open key means any key
14// <i> either stored in RAM (lifetime set to PSA_KEY_LIFETIME_VOLATILE), or
15// <i> used as part of a cryptographic operation.
16// <i> When using a key for a multi-part (setup/update/finish) operation, a key
17// <i> is considered to be open from the moment the operation is successfully
18// <i> setup, until it finishes or aborts.
19// <i> When an application tries to open more keys than this value accounts for,
20// <i> the PSA API may return PSA_ERROR_INSUFFICIENT_MEMORY. Keep in mind that
21// <i> other software included in the application (e.g. wireless protocol stacks)
22// <i> also can have a need to have open keys in PSA Crypto. This could lead to
23// <i> a race condition when the application key slot count is set too low for
24// <i> the actual usage of the application, as a software stack may not fail
25// <i> gracefully in case an application opens more than its declared amount of
26// <i> keys, thereby precluding the stack from functioning.
27// <i> Default: 4
28#define SL_PSA_KEY_USER_SLOT_COUNT (4)
29
30// <o SL_PSA_ITS_USER_MAX_FILES> PSA Maximum User Persistent Keys Count <0-1024>
31// <i> Maximum amount of keys (or other files) that can be stored persistently
32// <i> by the application through the PSA interface, when persistent storage
33// <i> support for PSA Crypto is included in the project.
34// <i> Due to caching logic, this setting does have an impact on static RAM usage.
35// <i> Note that this number is added to the potential requirements from other
36// <i> software components in the project, such that the total amount of keys
37// <i> which can be stored through the ITS backend can be higher than what is
38// <i> configured here.
39// <i>
40// <i> WARNING: When changing this setting on an application that is already
41// <i> deployed, and thus will get the change through an application upgrade,
42// <i> care should be taken to ensure that the setting is only ever increased,
43// <i> and never decreased. Decreasing this setting might cause previously
44// <i> stored keys/files to become inaccessible.
45// <i>
46// <i> It is not possible to change this setting when using V3 ITS Driver.
47// <i> The file-storage indexing is dependent on the maximum number of files,
48// <i> and if SL_PSA_ITS_USER_MAX_FILES is changed, ITS should be cleared and
49// <i> all files need to be stored again.
50// <i> Default: 128
51#define SL_PSA_ITS_USER_MAX_FILES (128)

54// <i> Devices that used PSA ITS together with gecko_sdk_3.1.x or earlier
55// <i> might have keys (or other files) stored in V1 format.

82// <i> WARNING: When using V3 driver, it is not possible to increase or decrease

115// <e SL_VSE_BUFFER_TRNG_DATA_DURING_SLEEP> Store already-generated random bytes before putting the device to sleep
116// <i> Using the hardware TRNG (for example through psa_generate_random()) will
117// <i> consume a non-negligible amount of power. A start-up routine must pass
118// <i> and a relatively large minimum amount of random bytes will be generated.
119// <i> Use cases where the device is frequently entering EM2/EM3 and thereafter
120// <i> consumes a small amount of data from the TRNG may benefit from buffering
121// <i> the existing random bytes before putting the device to sleep. These
122// <i> buffered bytes are then consumed until exhaustion before the TRNG needs
123// <i> to be initialized and used again.
124// <i>
125// <i> NOTE: this configuration option is only applicable for devices with a
126// <i> Virtual Secure Engine (VSE), and requires the 'Power Manager' component
127// <i> to be included in the project.
128// <i>
129// <i> Default: 0
130#define SL_VSE_BUFFER_TRNG_DATA_DURING_SLEEP (0)
131
132// <o SL_VSE_MAX_TRNG_WORDS_BUFFERED_DURING_SLEEP> Number of random words to buffer before putting the device to sleep <1-63>
133// <i> This option can be used to decrease the amount of random words that
134// <i> (if enabled) are buffered before the device enters EM2/EM3. Lowering this
135// <i> number will result in less static RAM usage, but also means that the TRNG
136// <i> potentially has to be initialized more times--leading to increased power
137// <i> consumption. By default this option in configured to buffer as much TRNG
138// <i> data as possible (limited by the depth of the TRNG FIFO).
139// <i>
140// <i> NOTE: this configuration option is only applicable when
141// <i> SL_VSE_BUFFER_TRNG_DATA_DURING_SLEEP is enabled.
142// <i>
143// <i> Default: 63
144#define SL_VSE_MAX_TRNG_WORDS_BUFFERED_DURING_SLEEP (63)
