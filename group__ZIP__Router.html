Events for the ZIP_Router.

Enumerator
ZIP_EVENT_RESET 

The Z/IP Gateway will reset itself upon receiving this event.

ZIP_EVENT_TUNNEL_READY 

Emitted when the portal tunnel has been established.

ZIP_EVENT_NODE_IPV4_ASSIGNED 

Emitted when a node has got a new ip address, the data argument contains the nodeid

ZIP_EVENT_NODE_DHCP_TIMEOUT 

Emitted when the DHCP process has given up trying to acquire an IP address. The data argument contains the node id

ZIP_EVENT_SEND_DONE 

Emitted when we are done sending an ip datagram over the LAN

ZIP_EVENT_NEW_NETWORK 

Emitted when the Z/IP Gateway has entered a new network.

ZIP_EVENT_QUEUE_UPDATED 

Node queue has been updated

ZIP_EVENT_ALL_NODES_PROBED 

Resource Directory has finished node probes.

All nodes in the resource directory is in one of the final states and RD is not locked.

ZIP_EVENT_BRIDGE_INITIALIZED 

Bridge has been initialized.

Allocation of virtual nodes may have been completed, not needed, or timed out.

ZIP_EVENT_ALL_IPV4_ASSIGNED 

The DHCP discover/renew round has completed, we have acquired an ip address for all nodes or timed out trying to acquire the address.

ZIP_EVENT_COMPONENT_DONE 

Event triggered by CC_PowerLevel, Mailbox, and node queues when they become idle.

Not used by, eg, Network Management, since there is a separate event for that.

ZIP_Router can then check if a backup is requested and, if all other components are idle, start the backup.

ZIP_EVENT_BACKUP_REQUEST 

Event triggered from signal handler when the zipgateway should back up as soon as it is idle.

ZIP_EVENT_NETWORK_MANAGEMENT_DONE 

Network Management has completed a request and NMS state has returned to NM_IDLE.

ZIP_EVENT_NM_VIRT_NODE_REMOVE_DONE 

Event to trigger reply to node removal frame indication completion of all associations and virtual node removal related to that node is done

ZIP_EVENT_NODE_PROBED 

Event emitted by Z/IP Resource Directory when it has completed a node probe (ending in STATUS_PROBE_FAIL or STATUS_DONE).
